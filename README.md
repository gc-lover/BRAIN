Репозиторий для хранения идей, концепций и документов по проекту NECPGAME.

## Описание

Репозиторий `.BRAIN` служит централизованным хранилищем для всех идей, концепций и заданий для AI агентов Cursor. Здесь формируется видение проекта, описываются игровые механики и создаются технические задания.

## О проекте

**NECPGAME** - это MMORPG с элементами киберспортивного шутера, экстрактшутера и глубокой социальной структурой в стиле CYBERPUNK 2077.

### Жанры
- MMORPG
- Киберспортивный шутер
- Экстрактшутер
- Социальная игра

### Сеттинг
- **Стиль**: CYBERPUNK 2077
- **Лор**: CYBERPUNK

### Источники вдохновения

- **BALDUR GATE 3** - глубокая ролевая система и система ветвления мира. Огромная реиграбельность. Возможность делать выбор, который кардинально влияет на сюжет. Романтические отношения с персонажами.

- **CYBERPUNK 2077** - киберпанк атмосфера и лор. В этой вселенной будет игра. Механики мира и основной игры.

- **VALORANT** - киберспортивные элементы шутера. Супергеройский шутер с активными элементами. Навыки, которые появляются с экипировкой, имплантами или деревом прокачки.

- **TARKOV** - экстрактшутер механики.

- **ARC RAIDERS** - кооперативная игра в киберпанк сеттинге. Экстрактшутер механики. Элементы PvE со сложными врагами. Сложный PvE интеллект у мобов.

- **MINECRAFT** - свобода творчества и строительства. Гибкая система строительства, крафта, модинг инструменты.

- **WOW** - MMORPG масштаб и социальные механики. Глубокий лор, мировые события, расовые события, клановые события и т.д. Рейдовая система.

- **LINEAGE** - клановая система и PvP. Механики заточки, фентезийная стилистика.

- **EVE online** - экономика и масштабные конфликты. Глобальная экономика, политика и торговля. Огромные кланы и т.д.

- **KENSHI** - открытый мир и свобода действий. Мир, который живёт сам по себе. Сложная и глубокая прокачка навыков через боль. Мир живёт самостоятельно. Происходят глобальные события без игрока. Мир сам себя живёт.

- **The Witcher 3 Wild Hunt** - повествование и квесты. Глубокий лор. Возможность взаимодействовать с большим количеством персонажей. Романтические отношения с персонажами. Глубокая система зачарований и эффектов.

- **RIMWORLD** - социальные взаимодействия и события. Мир живет сам по себе. События происходят и без игрока. Глобальная экономика, торговля, взаимодействия фракций. Возможность модинга.

- **MEGABONK** - динамичный геймплей. Глобальный рейтинг, простая и доступная рейтинговая система.

- **OBLIVION** - RPG составляющая, магия, фракции, сюжеты академий магов, воров, убийц и т.д. Сюжет и RPG, живой мир.

- **SKYRIM** - RPG составляющая, магия, фракции, сюжеты академий магов, воров, убийц и т.д. Сюжет и RPG, живой мир.

## Структура репозитория

Репозиторий организован по следующей архитектуре:

### 01-concepts/
Базовые концепции и видение проекта. Философия дизайна и основные столпы игры.

### 02-gameplay/
Игровые механики:
- **combat/** - Боевая система (шутер, экстрактшутер, способности, импланты)
- **progression/** - Прокачка и развитие (навыки, уровни, экипировка)
- **economy/** - Экономика (торговля, крафт, валюты)
- **social/** - Социальные механики (кланы, отношения, PvP)
- **world/** - Игровой мир (события, рейды, строительство)

### 03-lore/
Лор и сеттинг в стиле CYBERPUNK:
- **factions/** - Фракции (корпорации, банды, организации)
- **locations/** - Локации (города, зоны)
- **characters/** - Персонажи (NPC, протагонисты)

### 04-narrative/
Повествование и литературная часть:
- **scenarios/** - Сценарии (начало игры, события, взаимодействия, ветвление)
- **quests/** - Квесты (основные, побочные, фракций, событий, игроков)
- **lore/** - Лор игры (история вселенной, городов, фракций, событий, мифология)
- **npc-lore/** - Лор NPC (важные NPC, обычные NPC, враги, истории, отношения)
- **events-lore/** - Лор событий (мировые, фракций, локаций, динамические)
- **stories/** - Истории и повествования (персонажей, фракций, локаций, рассказы)
- **dialogues/** - Диалоги (NPC, квестов, событий, ветвление, романтические)

### 05-technical/
Технические задания:

## Цепочка взаимодействий

1. Идеи создаются и документируются в `.BRAIN`
2. **Проверка готовности:** Документы проверяются на готовность к созданию API задач (см. `06-tasks/config/readiness-check-guide.md`)
3. На основе готовых идей создаются API спецификации в `API-SWAGGER`
4. Бекенд и фронтенд реализуют функциональность на основе API

## Система проверки готовности документов

Документы в `.BRAIN` проверяются на готовность к созданию API задач для API-SWAGGER.

**Система проверки:**
- **Инструкция:** `06-tasks/config/readiness-check-guide.md` - подробная инструкция проверки готовности
- **Трекер:** `06-tasks/config/readiness-tracker.yaml` - отслеживание статусов готовности документов
- **Правила:** `.cursor/rules/brain-readiness-checker-rules.mdc` - правила для AI агента-проверяющего

**Статусы готовности:**
- `ready` - документ готов к созданию задачи API
- `needs-work` - нужна доработка документа
- `blocked` - заблокирован зависимостями
- `in-review` - проверяется на готовность
- `not-applicable` - документ не предназначен для создания API

**Критерии готовности:**
- Статус документа: `approved` или `review`
- Полнота информации: достаточно деталей для создания API
- Отсутствие блокирующих TODO
- Конкретность: документ содержит конкретные механики/концепции
- Структурированность: документ понятен и логичен

Подробнее см. [Инструкцию проверки готовности](./06-tasks/config/readiness-check-guide.md)
