# .BRAIN CORE

## Назначение
- Подготовка и поддержание идей, концепций, лора и технических описаний.
- Управление статусами документов (`draft/review/approved`, `api-readiness`), трекерами (`readiness-tracker.yaml`, `implementation-tracker.yaml`).
- Передача готовых материалов агентам API Task Creator и API Executor.

## Главные ссылки
- Общие правила: `../GLOBAL-RULES.md`
- Сценарий агента: `МЕНЕДЖЕР.MD`
- Трекинг: `06-tasks/config/readiness-tracker.yaml`, `06-tasks/config/implementation-tracker.yaml`
- Шаблоны и примеры: `МЕНЕДЖЕР-PROCEDURES.md`, `МЕНЕДЖЕР-CHECKLIST.md`, `МЕНЕДЖЕР-EXAMPLES.md`

## Ключевые обязанности
1. Формировать документы, соблюдая лимит 500 строк и общие шаблоны.
2. Указывать целевой микросервис/модуль для каждой фичи (см. таблицу в `GLOBAL-RULES.md`).
3. Обновлять метаданные (`api-readiness`, `Last Updated`, версии) и соответствующие трекеры.
4. Поддерживать структуру каталогов (`01-concepts`, `02-gameplay`, …) и актуальность ссылок на API спецификации.
5. Запускать Clear-процедуры (через `КЛИР.MD`) после завершения полного цикла реализации.

## Шаблоны и ограничения
- Перед серийным созданием материалов убеждайся, что существует актуальный шаблон. При необходимости создай/обнови его в разделе `06-tasks/ideas` или профильной директории.
- При превышении лимита строк разбивай документ на части (`*_0001.md`, `*_0002.md`) и добавляй навигацию.

## Интеграция с другими репозиториями
- Связывай документы с OpenAPI заданиями через `API-SWAGGER/tasks/config/brain-mapping.yaml`.
- После завершения backend/frontend стадий обновляй `implementation-tracker.yaml` по процедурам БЭКТАСК/ФРОНТТАСК.
- Весь контент, влияющий на OpenAPI, должен явно описывать микросервис и предполагаемые доменные каталоги (`api/v1/<microservice>/<domain>/`).

## Контроль качества
- Регулярно пересматривай документы на предмет устаревшей информации и инициируй очистку.
- Используй реальные временные метки (через `powershell -Command "Get-Date -Format 'yyyy-MM-dd HH:mm'"`).
- Все изменения документируй и коммить через `scripts/autocommit.ps1`.
