## ОБЯЗАТЕЛЬНАЯ процедура: Получение реального времени

**НИКОГДА НЕ ВЫДУМЫВАЙ ДАТЫ И ВРЕМЯ!**

При обновлении метаданных документов ВСЕГДА используй реальное время из системы:

### Windows:
```powershell
powershell -Command "Get-Date -Format 'yyyy-MM-dd HH:mm'"
```

### Linux/Mac:
```bash
date '+%Y-%m-%d %H:%M'
```

### Когда использовать:
- При добавлении/обновлении `api-readiness-check-date` в документах
- При добавлении/обновлении `checked` в readiness-tracker.yaml
- При добавлении/обновлении `Last Updated` в документах
- При любых метаданных, связанных с временем

### Пример использования в процессе:
```
1. Запустить команду получения времени
2. Получить результат (например: 2025-11-06 20:25)
3. Использовать это время для всех обновлений в текущей сессии
4. НЕ выдумывать другое время
```

## ОБЯЗАТЕЛЬНАЯ процедура: Обновление метаданных документов

**При проверке готовности документа ОБЯЗАТЕЛЬНО:**

### 1. Добавить/обновить метаданные в начале документа:

```markdown
**api-readiness:** ready | needs-work | blocked | in-review | not-applicable
**api-readiness-check-date:** YYYY-MM-DD HH:MM (РЕАЛЬНОЕ время!)
**api-readiness-notes:** Краткое описание статуса/проблем
```

### 2. Обновить readiness-tracker.yaml:

- Добавить новую запись или обновить существующую
- Использовать то же реальное время, что и в документе
- Заполнить все поля: path, status, version, checked, checker, notes, issues, api_target

### 3. Проверить корректность:

- Метаданные соответствуют реальному состоянию документа
- Время везде одинаковое (из системы)
- Все обязательные поля заполнены

## ОБЯЗАТЕЛЬНАЯ процедура: Коммиты изменений

**ПОСЛЕ ЗАВЕРШЕНИЯ ЛЮБОЙ РАБОТЫ ОБЯЗАТЕЛЬНО ДЕЛАТЬ КОММИТ!**

### Что коммитить:

- Все измененные файлы
- Обновленные метаданные документов
- Обновленный readiness-tracker.yaml
- Новые идеи в 06-tasks/ideas/
- Обновленные TODO.md, CURRENT-WORK статусы

### Как делать коммит:

```bash
# Windows
cd .BRAIN
.\scripts\autocommit.ps1 "Краткое описание работы"

# Linux/Mac
cd .BRAIN
./scripts/autocommit.sh "Краткое описание работы"
```

### Если сообщение не указано:

Скрипт сгенерирует сообщение автоматически на основе измененных файлов.

### Примеры сообщений коммитов:

```
"docs: проверка готовности документов боевой системы"
"docs: записаны идеи по системе крафта имплантов"
"docs: обновлен статус текущей работы и открытые вопросы"
"chore: массовая проверка готовности всех документов .BRAIN"
```

### ВАЖНО:

- Делать коммиты логическими блоками
- Не накапливать изменения
- После каждой части работы - коммит
- Не завершать работу без коммита
