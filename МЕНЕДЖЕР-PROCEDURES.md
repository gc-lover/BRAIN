# Процедуры Brain Manager Agent

## Ссылки

- `.BRAIN/CORE.md` — структура репозитория, трекеры, шаблоны.

--

## 1. Получение реального времени
- Всегда используй системное время при обновлении метаданных.
- Команды:  
  - Windows: `powershell -Command "Get-Date -Format 'yyyy-MM-dd HH:mm'"`  
  - Linux/Mac: `date '+%Y-%m-%d %H:%M'`
- Одно значение времени применяется ко всем связанным изменениям в текущей сессии.

--

## 2. Обновление метаданных документов
1. В начале файла добавь/обнови блок:  
   `api-readiness`, `api-readiness-check-date`, `api-readiness-notes`, `version`, `last updated` (если предусмотрено).  
2. Убедись, что документ ≤ 500 строк; при необходимости создай продолжение (`*_0002.md`).  
3. Проверь наличие ссылок на целевой микросервис, доменный каталог OpenAPI и фронтенд-модуль (см. `GLOBAL-RULES.md`).  
4. При статусе `ready` добавь запись в `brain-mapping.yaml`; при любом статусе обнови `readiness-tracker.yaml`.

--

## 3. Работа с трекерами
1. `readiness-tracker.yaml` — обнови поля: `path`, `status`, `version`, `checked`, `checker`, `notes`, `api_target`.  
2. `brain-mapping.yaml` — фиксируй связи документ → API → task (следи за лимитом строк, используй `_000X`-продолжения).  
3. `implementation-tracker.yaml` — обновляй только при согласовании с БЭК/ФРОНТ агентами.  
4. После каждого обновления убедись, что нет «битых» ссылок и сохранена сортировка/структура файла.

--

## 4. Шаблоны и идеи
1. Новые идеи, исследования, вопросы оформляй через актуальные шаблоны (`IDEA-TEMPLATE.md`, `TEMPLATES-GUIDE.md`).  
2. Всегда указывай: целевой микросервис, каталог `api/v1/<domain>/...`, фронтенд модуль, связанные документы.  
3. README/индексы обновляй сразу после добавления файла.  
4. Все вспомогательные материалы храни в пределах лимита 500 строк.

--

## 5. Коммиты
1. После каждого логического блока работы выполняй `scripts/autocommit.ps1|.sh` из корня репозитория.  
2. Убедись, что в коммит попали: обновлённые документы, трекеры, README/индексы.  
3. Сообщение коммита отражает суть изменений (например, `docs: readiness audit for shooter mechanics`).  
4. Перед запуском скрипта проверь, что нет незапланированных файлов или изменений.
