# Аномальные пасхальные сценарии — рабочий документ

## 1. Задачи
- Сформировать календарь редких аномалий с привязкой к погоде, времени и активности.
- Описать зависимости между `world-service`, `social-service`, `auth-service`, `inventory` и `gameplay-service`.
- Подготовить требования к визуальным эффектам и уведомлениям.
- Определить метрики мониторинга и анти-абьюз.

## 2. Общая концепция
- Аномалии создают «чизовые» моменты (площадь спектров, река, башня, архив, шторм, станция).
- Проведение аномалий должно быть редким, чтобы сохранить уникальность и избежать перегрузки сервера.
- Требуется возможность проводить GM-события вручную для тестов.

## 3. План работ
1. **Календарь и расписание**
   - Описать CRON/таймеры: площать (каждый вторник 21:00), река (полнолуние), башня (активность > X), шторм (случайный шанс), станция (по запросу).
   - Продумать fallback сценарии при пропуске аномалии.
2. **Технический дизайн**
   - ERD таблиц: `world.anomaly_schedule`, `world.anomaly_events`, `social.anomaly_participants`.
   - API: `GET /world/anomalies/upcoming`, `POST /world/anomalies/trigger` (internal).
3. **Визуал и UX**
   - Мокап визуального слоя для каждой аномалии.
   - Уведомления (HUD + почта) с вариантами «opt-out».
   - Адаптация для low-graphics режимов.
4. **Ревью риска**
   - Проверка нагрузки (особенно `Башня резонансных часов`).
   - Анти-абьюз жетонов/баффов.

## 4. Метрики и мониторинг
- Количество запусков каждой аномалии.
- Среднее число участников и участниц.
- Время ответа серверов при активации (latency spikes).
- Кол-во предупреждений анти-абьюза.

## 5. Открытые вопросы
- Нужно ли позволять гильдиям заранее бронировать аномалии?
- Какие визуальные ограничения для игроков с эпилепсией?
- Требуется ли отдельная таблица наград для каждого события?

- Подготовить черновик календаря (2025-11-08–2026-01-01).
