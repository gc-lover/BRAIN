# Кросс-культурный атлас пасхалок — рабочий документ

## 1. Цели
- Детализировать сезонный контент (граффити, музыкальные сетки, рынок капсул, экскурсии, фестиваль, музей).
- Определить требования к экономике, социальным механикам и аудио.
- Подготовить UX-гайды и moodboard запросы.
- Согласовать план rollout на сезон (как минимум 3 месяца).

## 2. Ключевые элементы
- **Аллея граффити:** ежедневная ротация, фотомоды, фильтр `Dual Frame`.
- **Музыкальный павильон:** многослойные треки, синхронизация панелей.
- **Рынок капсул:** косметика + блюда, эмоция `World Pulse`.
- **Гид «Путешествие по шуму»:** маршрут по скрытым сценам, титул `Cartographer of Noise`.
- **Фестиваль «Слияние ветров»:** AR-сцены, браслеты, кат-сцена благодарности.
- **Музей «Коды дружбы»:** пользовательские экспозиции.

## 3. Технические требования
- **World-service:** расписания, координаты локаций, AR слои.
- **Social-service:** фото-шеринг, экскурсии, запись экспозиций.
- **Economy-service:** цены капсул, дроп браслетов, предметы.
- **Audio-service:** плейлисты «Metropolis», «Tropical», «Nordic» и др.
- **Inventory:** фильтр, эмоция, предметы (браслеты, «Сеть соединённых рук»).
- **Auth-service:** флаги доступа к пользовательским экспозициям.

## 4. UX / Art задачи
- Мокапы граффити (4 варианта), фестивальных сцен, музейных экспозиций.
- План фотомода (UI обновление, фильтр `Dual Frame`).
- Руководство по подаче мультикультурных элементов, без стереотипов.
- Таблица цветов/шрифтов для каждой зоны.

## 5. Экономика и награды
- Определить стоимость капсул, лимиты покупок.
- Расчёт ценности эмоции `World Pulse` и фильтра `Dual Frame`.
- Таблица обмена браслетов → кат-сцена.
- Потенциальные сезонные валюты? (решить на этапе дизайна экономики).

## 6. Мониторинг
- Посещаемость аллеи, рынка, музея.
- Количество собранных браслетов/капсул.
- Активность пользовательских экспозиций (создано / просмотрено / отмодерировано).
- Sentiment по соц-каналам (внутренняя аналитика).

## 7. Открытые вопросы
- Какой лимит на пользовательские экспозиции (по размеру/количеству)?
- Нужен ли opt-in на AR элементы для low-spec устройств?
- Требуется ли локализация уникальных названий (типа «Electro-Sakura»)?
