# Очереди статусов готовности документов

Единый пул статусов: `ready`, `needs-work`, `in-review`, `blocked`, `not-applicable`.

- Каждый статус хранится в отдельном файле `*.md` (лимит 500 строк).  
- При переполнении создавайте продолжения (`<status>_0001.md`, `<status>_0002.md`) и указывайте ссылку на следующую часть в начале файла.  
- При смене статуса переносите запись между файлами без дублирования.  
- Примеры оформления приведены внутри каждого файла.
