## Назначение

Эта директория содержит **архив всех завершенных работ** проекта NECPGAME. Управляется агентом **Clear Agent (КЛИР)**.

**Зачем нужен архив:**
- Освобождение активных трекеров от завершенных записей
- Сохранение истории всех завершенных фич
- Быстрый доступ к информации о прошлых работах
- Создание статистики и отчетов

---

## Структура архива

### 1. completed-features/

**Назначение:** Сводные отчеты о завершенных фичах

**Структура:**
```
completed-features/
├── README.md              # Описание структуры
├── 2025-11/               # Группировка по месяцам
│   ├── feature-reports-001.md  # Фичи 1-10 (400-500 строк)
│   ├── feature-reports-002.md  # Фичи 11-20
│   └── README.md          # Индекс отчетов месяца
└── 2025-12/
    └── ...
```

**Формат отчета:** См. [КЛИР.MD](../../КЛИР.MD#процедура-2-формирование-отчета-о-фиче)

**Формат архивных файлов:** См. [КЛИР.MD](../../КЛИР.MD#процедура-3-архивация-записи-из-трекера)

## Правила работы с архивом

### Для Clear Agent (КЛИР):

1. **Группировать по месяцам** - все архивы группируются по формату `YYYY-MM`
2. **Разбивать отчеты** - максимум 500 строк на файл
3. **Нумеровать файлы** - 001, 002, 003, ...
4. **Обновлять индексы** - после каждой архивации
5. **Создавать README** - в каждой новой директории

### Для других агентов:

1. **НЕ читать архив** без необходимости (большой объем)
2. **Использовать INDEX.md** для быстрого поиска
3. **Обращаться к архиву** только для контекста старых фич
4. **НЕ модифицировать архив** - только чтение

---

## Периодичность архивации

### Рекомендуемая:

- **Еженедельно:** Если разработка активная (>5 фич в неделю)
- **Ежемесячно:** Если разработка умеренная (<5 фич в неделю)
- **По требованию:** Когда трекеры стали слишком большими

### Триггеры для архивации:

1. **implementation-tracker.yaml > 50 записей** → срочная архивация
2. **brain-mapping.yaml > 100 записей** → срочная архивация
3. **TODO.md > 100 задач** → срочная очистка
4. **Конец месяца** → плановая архивация
