## Краткое описание

Этот документ описывает систему шаблонов документов в проекте NECPGAME.

**Основной принцип:** Перед массовым созданием однотипных документов **ОБЯЗАТЕЛЬНО создавать шаблон**.

---

## Когда создавать шаблоны

### ОБЯЗАТЕЛЬНО создавать шаблон:

1. **Перед массовым созданием однотипных документов**
   - Если планируется создать 3+ документа одного типа
   - Например: множество NPC, множество квестов, множество локаций

2. **Для повторяющихся структур**
   - Если структура документа будет повторяться
   - Например: шаблон описания оружия, шаблон описания навыка

3. **Для стандартизации формата**
   - Чтобы все документы одного типа имели одинаковую структуру
   - Улучшает читаемость и навигацию

### Можно не создавать шаблон:

1. **Уникальные документы**
   - Если документ создаётся в единственном экземпляре
   - Если структура специфична только для этого документа

2. **Экспериментальные документы**
   - При проработке новой концепции
   - Можно создать шаблон позже, если понадобится

## Где хранить шаблоны

### 1. В ARCHITECTURE.md (основное хранилище)

**Расположение:** `.BRAIN/ARCHITECTURE.md`

**Секция:** "Шаблоны документов"

**Формат:**
```markdown
## Шаблоны документов

### Шаблон NPC

Для создания описания NPC используй следующий шаблон:

\`\`\`markdown
# Имя NPC

**Статус:** draft | review | approved
**Версия:** 1.0.0
**Дата создания:** YYYY-MM-DD
...
\`\`\`
```

### 2. В соответствующих директориях

**Расположение:** В той же директории, где будут создаваться документы

**Имя файла:** `[ТИП]-TEMPLATE.md`

**Примеры:**
- `.BRAIN/04-narrative/npc-lore/NPC-TEMPLATE.md`
- `.BRAIN/04-narrative/quests/QUEST-TEMPLATE.md`
- `.BRAIN/03-lore/locations/LOCATION-TEMPLATE.md`

### 3. В 06-tasks/ideas/ (для идей)

**Расположение:** `.BRAIN/06-tasks/ideas/IDEA-TEMPLATE.md`

**Используется:** Для записи идей

## Как использовать шаблоны

### Процесс использования шаблона:

1. **Найти шаблон:**
   - Проверить ARCHITECTURE.md (секция "Шаблоны документов")
   - Или найти файл [ТИП]-TEMPLATE.md в нужной директории

2. **Скопировать шаблон:**
   - Скопировать содержимое шаблона полностью
   - Создать новый файл с правильным именем

3. **Заполнить шаблон:**
   - Заменить плейсхолдеры (например, `[Имя NPC]` → `Марко "Фикс" Санчес`)
   - Заполнить все секции
   - Удалить секции, которые не применимы (если указано в шаблоне)

4. **Сохранить документ:**
   - Сохранить файл в соответствующей директории
   - Убедиться, что все метаданные заполнены

5. **Обновить список шаблонов:**
   - Если создан новый шаблон, добавить его в ARCHITECTURE.md
   - Добавить в список ниже в этом документе

## Правила создания шаблонов

### ОБЯЗАТЕЛЬНЫЕ элементы любого шаблона:

1. **Метаданные (frontmatter или в начале документа):**
   ```markdown
   **Статус:** draft | review | approved
   **Версия:** 1.0.0
   **Дата создания:** YYYY-MM-DD
   **Приоритет:** низкий | средний | высокий (если применимо)
   ```

2. **Плейсхолдеры для замены:**
   - Используй квадратные скобки: `[Имя персонажа]`, `[Название локации]`
   - Или пиши "TODO: заполнить"

3. **Комментарии для пользователя:**
   - Объясняй, что нужно заполнить
   - Приводи примеры, если нужно

4. **Секция "История изменений":**
   - Для отслеживания версий документа

### Рекомендуемые элементы:

5. **Краткое описание** (1-2 предложения)
6. **Детальное описание** (подробности)
7. **Связи с другими документами/системами**
8. **Примеры** (если применимо)
9. **Заметки** (дополнительная информация)

### Форматирование:

- Используй Markdown для форматирования
- Структурируй документ с заголовками (##, ###)
- Используй списки для перечислений
- Используй таблицы для структурированных данных (если нужно)

## Процесс создания нового шаблона

### Шаг 1: Определить необходимость

Ответь на вопросы:
1. Будет ли создано 3+ документа этого типа?
2. Имеет ли смысл стандартизировать структуру?
3. Повторяется ли структура документа?

Если хотя бы на 2 вопроса ответ "да" - создавай шаблон.

### Шаг 2: Определить структуру

1. Проанализируй существующие документы этого типа (если есть)
2. Определи обязательные секции
3. Определи опциональные секции
4. Определи метаданные

### Шаг 3: Создать файл шаблона

1. Создай файл `[ТИП]-TEMPLATE.md` в соответствующей директории
2. Добавь метаданные (статус, версия, тип)
3. Добавь все секции с плейсхолдерами
4. Добавь комментарии для пользователя
5. Добавь примеры (если нужно)

### Шаг 4: Добавить в ARCHITECTURE.md

1. Открой `.BRAIN/ARCHITECTURE.md`
2. Найди секцию "Шаблоны документов"
3. Добавь описание нового шаблона:
   - Название
   - Расположение
   - Назначение
   - Краткое описание структуры

### Шаг 5: Добавить в этот документ

1. Открой `.BRAIN/06-tasks/config/TEMPLATES-GUIDE.md`
2. Найди секцию "Список существующих шаблонов"
3. Добавь новый шаблон с описанием

### Шаг 6: Протестировать шаблон

1. Используй шаблон для создания тестового документа
2. Проверь, что все секции понятны
3. Убедись, что ничего не упущено
4. При необходимости доработай шаблон

### Шаг 7: Сделать коммит

```powershell
.\scripts\autocommit.ps1 "docs: Добавлен шаблон для [тип документа]"
```

## Обновление существующих шаблонов

### Когда обновлять шаблон:

1. **Найдена недостающая секция**
   - Если при создании документов выясняется, что нужна дополнительная секция

2. **Изменилась структура**
   - Если изменились требования к документам этого типа

3. **Улучшение формата**
   - Если найден более удобный способ структурирования

### Процесс обновления:

1. Обнови файл шаблона ([ТИП]-TEMPLATE.md)
2. Увеличь версию шаблона
3. Добавь запись в "История изменений" шаблона
4. Обнови описание в ARCHITECTURE.md (если структура сильно изменилась)
5. Обнови этот документ (если нужно)
6. Сделай коммит

**⚠️ ВАЖНО:** Обновление шаблона **НЕ требует** обновления всех существующих документов этого типа. Существующие документы остаются как есть. Новые документы создаются по новому шаблону.

## Часто задаваемые вопросы

### 1. Обязательно ли использовать шаблоны?

**Обязательно:**
- При массовом создании однотипных документов (3+)
- Когда есть готовый шаблон для этого типа документа

**Опционально:**
- Для единичных уникальных документов
- Для экспериментальных концепций

---

### 2. Что делать, если шаблон не подходит?

1. Используй шаблон как основу
2. Адаптируй под свои нужды
3. Удали ненужные секции
4. Добавь необходимые секции
5. Если изменения значительные - подумай о создании нового шаблона

---

### 3. Куда добавлять новые шаблоны?

1. Создай файл в соответствующей директории: `[ТИП]-TEMPLATE.md`
2. Добавь описание в ARCHITECTURE.md (секция "Шаблоны документов")
3. Добавь в этот документ (секция "Список существующих шаблонов")

---

### 4. Нужно ли обновлять старые документы при изменении шаблона?

**Нет, не обязательно.** Обновление шаблона влияет только на новые документы.

Старые документы можно обновить по желанию, но это не требуется.

---

### 5. Как понять, что нужен новый шаблон?

Если ты создал 2+ документа с похожей структурой и планируешь создать ещё - пора создать шаблон.

**Признаки:**
- Копируешь структуру из предыдущего документа
- Документы одного типа имеют разную структуру (плохо для читаемости)
- Тратишь время на форматирование каждого нового документа
