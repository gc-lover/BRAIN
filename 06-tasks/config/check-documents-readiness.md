# Проверка готовности документов .BRAIN к созданию API задач

**Приложи этот файл агенту для запуска проверки документов.**

---

## Задача

Проверить все документы в `.BRAIN` на готовность к созданию API задач для `API-SWAGGER`.

## Что нужно сделать

1. **Сканировать все документы:**
   - Найти все `.md` файлы в `.BRAIN`
   - Игнорировать служебные файлы: `README.md`, `ARCHITECTURE.md` и другие служебные
   - Игнорировать файлы в: `.cursor/`, `scripts/`, `06-tasks/` (кроме `config/`)
   - Фокусироваться на файлах в:
     - `01-concepts/`
     - `02-gameplay/`
     - `03-lore/`
     - `04-narrative/`
     - `05-technical/`

2. **Применить критерии готовности:**
   - Использовать инструкцию: `.BRAIN/06-tasks/config/readiness-check-guide.md`
   - Использовать правила: `.BRAIN/.cursor/rules/brain-readiness-checker-rules.mdc`
   - Применять все критерии готовности к каждому документу

3. **Определить статус готовности:**
   - `ready` - документ готов к созданию задачи API
   - `needs-work` - нужна доработка документа
   - `blocked` - заблокирован зависимостями
   - `in-review` - проверяется на готовность
   - `not-applicable` - документ не предназначен для создания API

4. **Обновить метаданные документов:**
   - Добавить в начало каждого проверенного документа:
     ```markdown
     **api-readiness:** ready | needs-work | blocked | in-review | not-applicable
     **api-readiness-check-date:** YYYY-MM-DD HH:MM
     **api-readiness-notes:** Заметки о готовности/проблемах (опционально)
     ```

5. **Обновить трекер готовности:**
   - Файл: `.BRAIN/06-tasks/config/readiness-tracker.yaml`
   - Добавить/обновить запись для каждого проверенного документа:
     ```yaml
     - path: ".BRAIN/path/to/document.md"
       status: "ready" | "needs-work" | "blocked" | "in-review" | "not-applicable"
       version: "1.0.0"
       checked: "2025-11-02 15:30"
       checker: "AI Agent"
       notes: "Заметки о готовности"
       issues: []
       api_target: "api/v1/path/to/api.yaml" | null
     ```

6. **ОБЯЗАТЕЛЬНО: Сделать коммит всех изменений:**
   - Использовать скрипт autocommit: `.\scripts\autocommit.ps1 [сообщение]` или `./scripts/autocommit.sh [сообщение]`
   - Коммитить обновленные метаданные документов и readiness-tracker.yaml
   - Не завершать проверку без коммита изменений

## Критерии готовности (кратко)

- ✅ **Статус документа:** `approved` или `review` (не `draft`)
- ✅ **Полнота информации:** достаточно деталей для создания API спецификации
- ✅ **Отсутствие блокирующих TODO:** нет критичных TODO, которые блокируют создание API
- ✅ **Конкретность:** документ содержит конкретные механики/концепции (не слишком общий)
- ✅ **Структурированность:** документ понятен и логичен
- ✅ **Версионирование:** документ имеет версию (желательно)
- ✅ **Документация:** есть описание концепций, примеры (если применимо)

## Результат

После проверки:
1. Все документы проверены
2. Метаданные документов обновлены
3. `readiness-tracker.yaml` содержит все проверенные документы
4. Документы со статусом `ready` отмечены для создания API задач
5. **ОБЯЗАТЕЛЬНО: Сделан коммит всех изменений**

## Коммит изменений

**ВАЖНО:** После проверки документов **ОБЯЗАТЕЛЬНО** сделать коммит всех изменений.

**Что нужно закоммитить:**
- Обновленные метаданные документов (api-readiness, api-readiness-check-date, api-readiness-notes)
- Обновленный readiness-tracker.yaml со всеми проверенными документами

**Как сделать коммит:**
1. Использовать скрипт autocommit из корня репозитория `.BRAIN`:
   - Windows: `.\scripts\autocommit.ps1 [сообщение]`
   - Linux/Mac: `./scripts/autocommit.sh [сообщение]`
2. Если сообщение не указано, скрипт сгенерирует его автоматически
3. Скрипт автоматически сделает commit и push изменений

**Пример сообщения коммита:**
```
chore: проверка готовности документов .BRAIN к созданию API задач
```
