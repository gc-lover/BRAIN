## Назначение
Этот файл — напоминание для менеджера `.BRAIN` и связанных агентов, какие шаги необходимо выполнить перед передачей задачи в API Executor (`АПИТАСК`). Подробные шаги и требования описаны в:
- `GLOBAL-RULES.md`
- `.BRAIN/CORE.md`
- `API-SWAGGER/ДУАПИТАСК.MD`
- `API-SWAGGER/АПИТАСК.MD`
- `API-SWAGGER/CORE.md`

---

## Что нужно подготовить в `.BRAIN`
1. Документ с метаданными (`draft/review/approved`, `api-readiness`) и указанием микросервиса/домена.  
2. Ссылку на предполагаемый каталог OpenAPI (`api/v1/<microservice>/<domain>/feature.yaml`).  
3. Обновлённые трекеры: `06-tasks/config/readiness-tracker.yaml`, `06-tasks/active/CURRENT-WORK/current-status.md`.

---

## Передача в API-SWAGGER
- Создай задание (`ДУАПИТАСК`) с использованием шаблона `tasks/templates/api-generation-task-template.md`.  
- Добавь запись в `API-SWAGGER/tasks/config/brain-mapping.yaml` (либо актуальную часть `_000X.yaml`).  
- В исходном документе `.BRAIN` добавь секцию `API Tasks Status` с Task ID и датой.  
- Сообщи АПИТАСК, что задание готово.

---

## Контрольный список
- [ ] Документ `.BRAIN` имеет статус `api-readiness: ready` и актуальную дату проверки.

- [ ] Коммит выполнен через `scripts/autocommit.ps1|.sh`.
